PetalData

{{ page.description }}

Your cloud apps don't make it easy to download data for analysis. For every app you use, you must learn a new API, paginate over results, convert columns into proper datatypes, write scripts to fetch new data, and more.

PetalData solves this problem by providing a consistent API for downloading analysis-ready data across cloud apps. Start delivering insights in minutes.

Try these examples:

  • Stripe
  • Hubspot
{% highlight python %} import petaldata petaldata.datasets.stripe.api_key = "[YOUR STRIPE API KEY]" invoices = petaldata.datasets.stripe.Invoice() invoices.download(limit=5) # Access downloaded invoices via a Pandas Dataframe df = invoices.df {% endhighlight %}
{% highlight python %} import petaldata petaldata.datasets.hubspot.api_key = "[YOUR HUBSPOT API KEY]" contacts = petaldata.datasets.hubspot.Contacts() contacts.download(limit=5) # Access downloaded contacts via a Pandas Dataframe df = contacts.df {% endhighlight %}

No ETL pipelines. No data warehouse.

Running a full ETL (extract, transform, load) pipeline and data warehouse is expensive and time-consuming to maintain. If you aren't ready to make this investment (and aren't storing terrabytes of data in your cloud apps ... ahem - that's most of us) PetalData is your secret sauce. Data scientists and data hobbyists can quickly explore the data in the cloud apps you already use without assembling several expensive parts.

How PetalData Works

There are two components to PetalData: our Python library for downloading datasets and the PetalData server.

While we provide a vanilla HTTP API, you'll primarly interact with PetalData with the petaldata Python package. This library standardizes the process of downloading, saving, and updating datasets. Datasets can be stored locally on your computer or in Amazon S3.

The most brittle parts of the download processes are conducted on our servers, accessible at petaldata.app. The Python library sends requests to the petaldata server. Server-side, we contact the appropriate cloud app, fetch your data, format it, and stream it back to the client. Data exporting is a brittle process that requires continual debugging, tweaks, and maintenance. If we didn't have our servers perform the hard work, debugging issues and frequent release cycles would be painful for everyone.

Security

We're just as paranoid about leaking data as you are. All network traffic to and from PetalData is via HTTPS. PetalData does not store or log credentials for your cloud services on our servers.
